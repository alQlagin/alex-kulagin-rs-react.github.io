Component-Based

Since component logic is written in JavaScript instead of templates, you can easily pass rich data through your app and keep state out of the DOM

Function and Class Components

Function Components

The simplest way to define a component is to write a JavaScript function 
                        
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}
Function components are stateless

Class Components

                    
class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}
It may have local state 
                    
class Welcome extends React.Component {
  constructor(props) {
    super(props);
    this.state = {date: new Date()};
  }
  render() {
    return <div>
        <h1>Hello, {this.props.name}</h1>
        <span>Now {this.state.date.toLocaleTimeString()}</span>
    </div>
  }
}

Composing components

                    
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

function App() {
  return (
    <div>
      <Welcome name="Sara" />
      <Welcome name="Cahal" />
      <Welcome name="Edite" />
    </div>
  );
}

ReactDOM.render(
  <App />,
  document.getElementById('root')
);

Virtual DOM

A programming concept where an ideal, or “virtual”, representation of a UI is kept in memory and synced with the “real” DOM by a library such as ReactDOM. This process is called reconciliation
You tell React what state you want the UI to be in, and it makes sure the DOM matches that state
This abstracts out the attribute manipulation, event handling, and manual DOM updating that you would otherwise have to use to build your app
